# 页面 08：多图 OCR 排序确认流程（移动端）
## 背景与目标
- 在“创建文章”时支持一次上传多张图片（同属一篇文章）。
- 增加 AI 排序推理能力：基于多图 OCR 结果推断页面先后顺序。
- 在正式创建文章前，新增“排序确认”步骤，让用户人工确认并可手动调整。
- 交互必须优先适配手机单手操作，避免复杂表格和多列布局。

## 适用范围
- 仅针对“上传图片并 OCR 导入文章”场景。
- 单图上传维持现有流程，不进入排序确认步骤。
- 多图上传（>=2）进入“AI 推理排序 -> 人工确认 -> 生成文章”流程。

## 流程总览
1. 用户在编辑页选择多张图片。
2. 前端调用批量 OCR 解析接口，后端返回每张图的 OCR 文本与元数据。
3. 后端调用 AI 排序推理 Prompt，返回建议顺序和置信度。
4. 前端进入“排序确认页”（移动端全屏），按 AI 建议顺序展示。
5. 用户可拖拽重排，确认后生成最终拼接文本。
6. 前端沿用现有创建文章接口提交文本，进入练习流程。

## AI Prompt：多图排序推理设计
### 输入信号（每张图片）
- `image_id`
- `filename`（如 `page_01.jpg`）
- `captured_at`（EXIF，可空）
- `ocr_text_head`（前 120-200 字）
- `ocr_text_tail`（后 120-200 字）
- `page_marker_candidates`（从 OCR 文本提取：第X页、Page X、Chapter、Section 等）
- `quality_flags`（OCR 质量、是否缺字、是否疑似封面/目录）

### 输出结构（严格 JSON）
```json
{
  "ordered_image_ids": ["img3", "img1", "img2"],
  "overall_confidence": 0.82,
  "reasoning_signals": [
    "filename_natural_order",
    "page_marker_match",
    "text_continuity_between_pages"
  ],
  "uncertain_pairs": [
    { "left": "img1", "right": "img2", "confidence": 0.55 }
  ],
  "warnings": ["Possible missing page between img1 and img2"]
}
```

### Prompt 规则要点
- 优先依据：页码线索 > 文本连续性 > 文件名自然序 > EXIF 时间。
- 输出必须可解释（`reasoning_signals` + `uncertain_pairs`）。
- 当置信度低或冲突高时，不做强结论，明确给 `warnings`。
- 禁止输出 markdown，禁止输出非 JSON 文本。

## 接口建议（文档阶段）
### 1）批量解析与排序推理
- `POST /api/bff/v1/articles/parse-upload-batch`
- 入参：`files[]`、`language_hint`（可选）
- 出参：
  - 每张图片 OCR 结果（文本、质量信息）
  - `order_suggestion`（AI 建议顺序）
  - `need_manual_confirm`（默认 `true`，多图场景强制人工确认）

### 2）创建文章（复用现有）
- 维持 `POST /api/bff/v1/articles`
- 前端将用户确认后的有序文本拼接后提交。
- 可选扩展：提交 `ordered_image_ids` 作为审计信息。

## 移动端交互设计（重点）
## 1. 页面形态
- 采用全屏“排序确认”页，不与上传入口混杂在同屏。
- 顶部：返回、标题“确认页面顺序”、简短提示。
- 中部：单列卡片列表（每张图片一张卡片）。
- 底部：固定操作栏（取消、确认顺序并生成文章）。

## 2. 卡片信息
- 缩略图（左）+ 主信息（右）：
  - 当前顺序号（1,2,3...）
  - 文件名
  - OCR 文本摘要（2-3 行）
  - 识别到的页码线索（如有）
- 右侧拖拽把手图标，支持长按拖拽排序。

## 3. 交互规则
- 默认按 AI 建议顺序展示。
- 用户长按卡片拖拽可调整顺序。
- 拖拽时列表自动滚动（处理长列表）。
- 提供“恢复 AI 建议顺序”按钮，便于快速回退。
- 当 AI 置信度低时，顶部展示黄色提示条，强调请人工核对。

## 4. 性能与可用性
- 图片多时（如 >20）优先展示缩略图 + 摘要，按需懒加载原图。
- OCR 文本只展示摘要，避免移动端滚动过长。
- 按钮区固定在底部，确保单手可点击。
- 关键操作（确认生成）前不再弹二次阻塞确认，保持流程连贯。

## 异常与兜底
- AI 推理失败：回退“文件名自然排序”并提示用户手动确认。
- OCR 空文本图片：仍保留卡片，但标记“文本识别较弱”。
- 疑似缺页：显示警告，不阻断创建；由用户决定继续。
- 用户未调整顺序也必须点击“确认顺序并生成文章”才进入创建。

## 验收标准
- 多图上传时必经排序确认步骤。
- 列表默认顺序来自 AI 建议结果。
- 用户可在手机上顺畅完成拖拽调整和确认。
- 最终文章文本顺序与确认列表一致。
- 低置信度或冲突场景有明确提示与人工兜底。

## 非目标（本期不做）
- 自动跳过人工确认（即使 AI 置信度高也不跳过）。
- 复杂文档结构识别（目录重排、跨章节重建）。
- 对 PDF 直接分页解析（本期仅图片文件）。
