# 页面 04：结果页

## 页面功能
- 承接一次练习提交后的结果展示。
- 默认强调详细对齐信息，帮助用户定位问题。
- 提供本等级整篇进度的简版矩阵视图。
- 由用户手动决定下一步（再练一次或进入下一段）。

## 页面布局
- 头部仅保留基础操作，不展示当前位置提示。
- 主体详情区（默认展开）：
  - 本次分数、达标状态（是否 `>= 85%`）
  - 当前等级与段落信息（放在结果详情区）
  - token 级对齐明细（B 方案：分离展示原文和识别文本）
    - 上行：原文（ref_tokens）
    - 下行：识别文本（hyp_tokens）
    - 状态标记：correct（绿色）、substitute（红色删除线）、missing（黄色背景）、insert（灰色）
  - 噪声块处理：
    - 可折叠展开/收起
    - 视觉标识：灰色背景 + “重复片段”提示文字
    - 默认状态：折叠显示
  - 差异类型与错误聚合
  - 可视化对齐结果（重点展示）
- 侧边/下方矩阵区：
  - 当前等级”段落达标矩阵”（简版）
  - 可查看各段是否已达标/跳过
- 底部动作区：
  - 未达标默认主按钮：`再练一次`
  - 达标后可见：`进入下一段`（手动触发）

## 交互方式
- 结果展示：
  - 页面进入即加载本次练习结果与当前等级矩阵。
- 未达标路径：
  - 点击 `再练一次` 返回练习页当前段继续训练。
- 达标路径：
  - 点击 `进入下一段` 前进到下一段练习。
  - 不自动跳转，保持用户可控。
- 矩阵交互：
  - 支持查看段落状态；是否允许直接点跳由流程策略控制（默认不破坏强引导）。
