# 强引导流程 V2 最终方案

更新时间：2026-03-04  
状态：已确认（基于问答记录固化）  
来源：`apps/guided_flow_v2/GUIDED_FLOW_QA.md`

## 1. 目标与范围
- 用户不再可在“首页 / 练习 / 进度 / 我的”间自由跳转。
- 引导强调“流程连续性”，同时保留必要用户自主性（如手动切等级、跳过段落）。

## 2. 核心原则
- 强引导：移除底部 Tab，全流程按步骤推进。
- 弱呈现：不使用强侵入式“上一步/下一步”大按钮；由页面内动作驱动流转。
- 去位置提示：页面顶部不展示“当前所处步骤/位置”说明。
- 可恢复：中断后自动恢复到最近未完成步骤。
- 可切换文章：从“我的”可随时进入历史文章并恢复其进度。

## 3. 信息架构
- 主流程页面
  - 动态入口页（仅历史用户出现）
  - 文章导入与全屏文本校对页（合并语种确认与分段确认）
  - 练习页（L1-L4 统一等级推进）
  - 结果页
- 辅助页面/入口
  - “我的”独立页面（全程可进入）
  - “进度”右上角轻入口（仅练习页可见，抽屉/弹层，不离开当前步骤）

## 4. 页面流转（文字版）
1. 登录后进入：
- 有历史进度：进入开始页
- 无历史进度：直达文章导入与全屏文本校对页

2. 开始页（历史用户）
- 仅显示：最近文章标题、当前等级、当前段、`继续上次`、`新建文章`

3. 文章导入与全屏文本校对页
- 主引导入口：粘贴板导入、文件导入（`txt/md`、图片 OCR）
- 弱引导入口：手动输入
- 默认不展示全屏文本框；触发任一输入方式后，打开顶层全屏文本框
- 全屏文本框以遮罩态显示，遮挡其他元素
- 三种来源统一汇入同一个全屏文本框
- 用户手动校对后进入下一步

4. 练习页
- 按当前文章、当前等级、当前段执行练习
- 未达标可重练或跳过（跳过会标记未完成）
- 开始录音后进入全屏专注遮罩（顶层覆盖），聚焦当前练习内容
- 结束录音后退出遮罩，回到练习页继续识别与提交流程

5. 结果页
- 默认强调详细对齐结果
- 展示简版“当前等级整篇达标矩阵”
- 达标后由用户手动进入下一段（不自动跳转）

6. 结束与继续
- 本等级所有段落达标后升级等级
- L4 完成后该文章主流程结束
- 用户可切到“我的”打开其他历史文章继续

## 5. 页面结构约束
- 统一要求：页面顶部不展示“当前所处步骤/位置”说明。

### 5.1 开始页（仅历史用户）
- 轻量信息块：
  - 最近文章标题
  - 当前等级（Lx）
  - 当前段
  - 操作：`继续上次`、`新建文章`

### 5.2 全屏文本校对页
- 输入来源：粘贴板 / 文件识别 / 手动输入
- 显示规则：默认不显示大文本编辑框；仅在触发任一输入方式后展示
- 层级规则：大文本编辑框以全屏顶层遮罩显示，遮挡其他元素
- 分段规则：以换行为段落边界，用户可在文本框内手动换行调整
- 语种规则：自动检测 + 用户可手动修改最终语种
- 下一步校验：
  - 文本非空
  - 段落数 >= 1
  - 语种已确定
  - 文本长度 >= 20 字符
- 段落上限：
  - 最大 30 段
  - 超出时阻断下一步，提示用户在本页手动拆分/精简

### 5.3 练习页
- 默认精简模式，支持展开完整模式
- 开始录音后展示全屏专注遮罩：顶层覆盖、遮挡其他元素，包含录音中状态、REC 指示、当前段落文本和结束录音主按钮
- 录音专注遮罩结构：
  - 顶部：关闭/取消、`录音中`状态、`REC`指示
  - 中部：当前段落练习文本（专注跟读）
  - 底部：`结束录音`主按钮
- 录音专注遮罩交互：
  - 点击`开始录音`进入遮罩并禁用底层交互
  - 点击`结束录音`退出遮罩并进入识别流程
  - 点击`关闭/取消`退出遮罩，本次录音不提交
- 精简模式展示：
  - 当前段/当前等级
  - 目标阈值（>=85%）
  - 录音与提交主操作
  - 本次结果与可前进行为
- 完整模式展示：
  - token 级对齐
  - 错词/差异统计
  - 最近成绩趋势
  - 等级达标矩阵与跳过段
  - 同步状态细节

### 5.4 结果页
- 默认展示详细对齐
- 展示简版达标矩阵（可点段落）
- 未达标默认引导动作：`再练一次`
- 达标后进入下一段：用户手动触发

### 5.5 我的页
- 独立页面，全流程可进入
- 主视觉：历史文章列表（默认按最近练习时间倒序）
- 账号区弱化，仅保留：
  - 用户名
  - 修改密码入口

## 6. 练习规则（强引导核心）

### 6.1 段落与等级
- 段落达标阈值：单次成绩 `>=85%`
- 同一篇文章在同一轮中，所有段落统一等级（L1-L4）
- 等级推进顺序固定：`L1 -> L2 -> L3 -> L4`

### 6.2 等级完成定义
- 当前等级下，文章所有段落都至少有 1 次 `>=85%` 才算等级完成

### 6.3 跳过规则
- 允许跳过未达标段
- 跳过段必须在本等级结束前补齐达标
- 跳过不改变等级完成判定

### 6.4 手动切等级
- 允许手动升降级
- 但切换会重置当前等级进度
- 必须明确提示“将重置当前等级进度”

### 6.5 跨等级回退
- 升到下一等级后，不允许回退到已完成等级

### 6.6 进度重置
- 不允许“清空文章进度重练”（仅向前推进）

## 7. 恢复与同步策略

### 7.1 恢复策略
- 中断定义：关闭页、刷新、重登返回、网络中断后恢复等
- 恢复规则：自动恢复最近未完成步骤
- 恢复提示：静默恢复（不提示）
- 路由方式：使用 `push` 方式（保留浏览器返回能力）

### 7.2 多文章切换
- 新建文章时，旧文章进度保留
- 从“我的”打开历史文章时，加载该文章进度继续
- 每篇文章独立保存流程状态

### 7.3 本地与服务端冲突
- 服务端优先
- localStorage 仅作临时缓存
- 冲突时本地被服务端覆盖

### 7.4 实时保存触发
- 关键动作即刻保存：
  - 提交练习
  - 跳过段落
  - 手动切等级
  - 切换文章
  - 离开页面时 flush

### 7.5 保存失败处理
- 不阻断用户操作
- 不回滚前端状态
- 必须明确提示”未同步”
- 重试策略：后台自动重试（指数退避：1s, 2s, 4s, 8s, 16s, 32s，最多 6 次）+ 手动一键重试
- 未同步数据存储：
  - 存储位置：localStorage
  - 存储结构：包含 `id`, `type`, `payload`, `createdAt`, `retryCount`, `lastError`
- 用户手动重试：
  - 触发方式：顶部显示未同步徽章，点击后展示列表
  - UI 反馈：重试中显示 loading，成功后移除，失败显示错误信息

### 7.6 未同步离开与过期
- 允许离开当前文章，但弹窗提示”可能丢失未同步进度”
- 未同步本地保留 24 小时
- 过期后自动清理并提示
- 不提供导出未同步数据

### 7.7 localStorage 最小字段
- `flow_version`
- `current_step`
- `article_draft`
- `context_ids`（`article_id` / `segment_id` / `practice_session_id`）
- `updated_at`

## 8. 后端影响与待执行
- 需要后端数据库结构调整，支持按 `user_id + article_id` 的进度持久化。
- 详细待办见：
  - `apps/api/docs/GUIDED_FLOW_V2_TODO.md`

重点包括：
- 文章级进度表
- 段落级+等级进度表
- 练习尝试记录
- 进度读取/提交/跳过/等级切换接口

## 9. 非目标（本阶段不做）
- 不提供用户自行重置整篇文章进度
- 不恢复底部 Tab 的自由导航
- 不在强引导中引入复杂统计大盘主视图

## 10. 验收标准（V2 方案层）
- 用户不能在主流程中任意跳页
- 任意中断后可恢复到最近未完成位置
- 同一文章等级推进满足“全段达标”规则
- 历史文章可按文章维度恢复
- 同步失败时用户可继续操作，且有明确“未同步”提示
