# 引导流程 V2 问答记录

状态：问答进行中
日期：2026-03-04
范围：仅新增“强引导流程”方案文档，不改现有代码路径。

## 设计原则
- 用户不能在“首页 / 练习 / 进度 / 我的”之间自由跳转。
- 流程必须从入口开始，按步骤引导前进。
- 现有实现保持不变；V2 仅在新目录完成方案设计。
- 页面导航控件弱化展示：不做显著的统一“上一步 / 下一步”大按钮。
- 每个页面拥有自己的前进/返回逻辑与触发条件。

## 已确认决策

### Q1. V2 的全局导航策略
问题：在强引导模式下，是否完全移除底部 Tab 导航？

选项：
- A（推荐）：完全移除底部 Tab，仅保留“上一步 / 下一步”操作。
- B：保留底部 Tab，但除当前步骤外全部禁用。
- C：保留底部 Tab，点击非当前步骤时弹窗阻止并提示回主流程。

结论：A（已确认）
理由：需要强引导，避免用户偏航；并由页面内部逻辑控制前进与回退。

### Q2. 中断后重新进入时的入口策略
中断定义：关闭页面、刷新页面、重登返回、网络中断后恢复等未正常完成流程的场景。

问题：用户中断后再次进入应如何处理？

选项：
- A（推荐）：自动恢复到“最近未完成步骤”，继续引导；不允许跳到任意步骤。
- B：每次都从第 1 步重新开始，未完成进度全部丢弃。
- C：弹出“继续上次/重新开始”二选一，默认继续上次。

结论：A（已确认）
理由：使用 localStorage 记录流程状态，下次打开自动继续上次练习。

补充：localStorage 最小记录字段（草案）
- `flow_version`: 流程版本号（用于兼容迁移）
- `current_step`: 当前步骤标识
- `article_draft`: 文章输入草稿（标题/文本/source_type/language）
- `context_ids`: 关键上下文 ID（article_id/segment_id/practice_session_id）
- `updated_at`: 最后更新时间（用于过期和冲突处理）

### Q3. 结果页后的下一步与段落晋级条件
问题：当用户完成一次“练习执行 + 结果查看”后，下一步默认是什么？

选项：
- A（推荐）：进入“下一段继续练习”（同一文章内）。
- B：回到流程起点（文章输入页）开始下一轮。
- C：停留在结果页，由用户手动选择“继续下一段/新建文章”。

结论：A（已确认）
理由：保持训练连续性，减少无效跳转。

补充确认：是否需要达标后才进入下一段？
- 结论：是（已确认）
- 规则：单次准确率 `>= 85%` 才允许进入下一段；否则留在当前段继续练习。

### Q4. 遮盖等级推进策略
问题：4 级遮盖（L1-L4）在段落之间如何推进？

结论（已确认）：
- 同一篇文章在同一轮练习中，所有段落使用统一遮盖等级。
- 先完成当前等级在“整篇文章”的练习目标，再进入下一个等级。
- 等级推进顺序固定：L1 -> L2 -> L3 -> L4。

理由：保持难度节奏一致，减少用户在不同段落间的认知切换。

### Q5. “一个等级完成” 的判定口径
问题：进入下一等级前，如何定义“当前等级已完成”？

选项：
- A（推荐）：当前文章所有段落都至少有 1 次成绩 `>=85%`，即判定该等级完成。
- B：当前文章所有段落都至少有 2 次成绩 `>=85%`，才算完成。
- C：按文章平均分判定（例如平均 `>=85%`），不要求每段都达标。

结论：A（已确认）
理由：兼顾强引导与推进效率，避免训练周期过长。

### Q7. 已完成段落是否允许回看/重练

选项：
- A（推荐）：允许回看但不允许重练（只读结果）。

- C：不允许回看，严格只看当前段。

结论：B（已确认）
理由：满足复盘与补强需求，同时保持主流程推进稳定。

### Q8. 当前段落多次未达标时是否允许“跳过”
问题：如果用户在当前段多次练习仍未达到 85%，是否允许先跳过该段继续后续段？

选项：
- A（推荐）：不允许跳过，必须当前段达标后才能前进。
- B：允许跳过，但该段标记为“未完成”，本等级结束前必须补做。
- C：允许直接跳过，后续不再强制回补。

结论：B（已确认）
理由：允许用户先推进流程，降低卡关挫败；同时保持等级完成标准不放松。

补充约束（已确认）：
- 跳过仅改变“当前练习顺序”，不改变“等级完成判定”。
- 一个等级完成仍必须满足：该等级下文章所有段落至少 1 次成绩 `>=85%`。

### Q9. 跨等级回退策略
问题：升级到下一等级后，是否允许回退到已完成等级？

选项：
- A（推荐）：升级到下一等级后，不允许降回已完成等级。
- B：允许降回已完成等级练习，但不影响已解锁等级状态。
- C：允许降回已完成等级，且会锁回后续等级（需重新解锁）。

结论：A（已确认）
理由：保持引导主线稳定，避免用户反复切换造成节奏中断。

### Q12. 本地进度与服务端进度冲突时的优先级

### Q16. 关键动作保存失败时的处理策略

补充约束（已确认）：

- 不自动回滚已完成的前端操作状态。
